## Design System

- **Colors:** Custom palette (parchment, pine, sage, earth, charcoal, etc.) defined in `tailwind.config.js`
- **Fonts:** Playfair Display (serif headings), Source Sans 3 (body), IBM Plex Mono (labels/metadata)
- **Tokens:** Defined in `src/index.css` with semantic utilities (.page-shell, .section-space, .surface-card, .button-* variants)

## Architecture

- **Content:** Markdown files in `src/content/` + JSON in `src/data/`
- **Components:** Astro components in `src/components/`
- **Pages:** Astro file-based routing in `src/pages/`
- **Layout:** BaseLayout wraps all pages with Header + Footer
- **Client JS:** Minimal inline `<script>` tags for filtering (Publications, Methods pages only)
- **Styling:** Tailwind CSS with custom semantic layer

## Important Decisions

1. **Content portability is sacred.** All content must be in standard Markdown + JSON. Do NOT use Astro-specific features (Content Collections, `.astro` files for content) for content storage.
2. **Rebrand:** "Plant Systems Physiology Lab" → "Crop Physiology Lab"
3. **New pages:** Capabilities, Methods & Protocols
4. **Student recruitment:** Primary goal. Must be specific and actionable.
5. **Form handling:** Use Netlify Forms or Formspree. No custom backend.
6. **No CMS:** Content updated by editing Markdown/JSON files in git.

## Patterns

- Components use PascalCase filenames (e.g., `TeamCard.astro`)
- Content uses kebab-case (e.g., `hplc-sop.md`)
- Data uses camelCase keys in JSON
- Utilities use camelCase (e.g., `safeHref.js`)

## Testing

- No test suite currently exists
- Manual testing: verify builds, check responsive layout, test filters, verify form submission
- Accessibility: keyboard navigation, screen reader testing with NVDA/VoiceOver

## Deployment

- Build: `npm run build`
- Output: `dist/` directory with static HTML
- Deploy via git push (auto-build on Netlify/Vercel)
- No server configuration needed
